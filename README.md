# Voxelized_Railway_Ballast_Generation

This repository contains the Python implementation used to generate and assemble heterogeneous voxelized railway ballast models for ground-penetrating radar (GPR) simulations.

The workflow provides a reproducible framework for constructing realistic ballast geometries from 3D stone models, voxelizing them into modular sub-blocks, and assembling these sub-blocks into a larger heterogeneous ballast domain suitable for numerical modelling.

## Associated publication

This repository accompanies the manuscript:

**Numerical investigation of GPR performance over voxelized heterogeneous railway ballast: influence of layer composition and antenna frequency**

submitted to *Computers & Geosciences*.

## Reproducibility

This repository provides the workflow used to generate the voxelized ballast models described in the associated manuscript.

The results are reproducible provided that the same input geometries, parameters, and script settings are used. Because the placement and orientation of ballast particles are generated using randomized rigid-body transformations, outputs may vary between runs unless random seeds are explicitly fixed.

## Workflow overview

The workflow consists of two main stages.

### Step 1 — Generation of voxelized ballast sub-blocks (`Code_1.py`)

This script generates **147 independent voxelized ballast sub-blocks** using procedurally placed 3D stone geometries.

Main operations include:

- loading 3D stone geometries from `.obj` files
- applying stochastic rigid-body rotations
- systematic placement of stones within a voxel box
- overlap-controlled voxel assembly
- generation of per-layer outputs and reports
- export of HDF5 and PKL files

Each sub-block has dimensions:

- **0.1 m × 0.1 m × 0.1 m**

with voxel size:

- **0.002 m × 0.002 m × 0.002 m**

The vertical domain is divided into four z-levels:

- 0.000–0.025 m
- 0.025–0.050 m
- 0.050–0.075 m
- 0.075–0.100 m

#### Outputs from `Code_1.py`

Typical outputs include:

- voxelized ballast sub-blocks in HDF5 format
- PKL files containing voxel grids
- 2D projection images for each z-level
- 3D coloured voxel visualizations for each z-level
- a text report summarizing overlap and filled fraction statistics

### Step 2 — Assembly of the final heterogeneous ballast domain (`Code_2.py`)

This script loads the **147 voxelized ballast sub-blocks** generated by `Code_1.py` and assembles them into a structured:

- **7 × 7 × 3**

layout to construct the final heterogeneous ballast domain.

Final domain size:

- **0.7 m × 0.7 m × 0.3 m**

Main operations include:

- loading individual HDF5 voxel blocks
- systematic 3D grid assembly
- boundary correction across internal interfaces
- 2D projection visualization
- export of the combined HDF5 volume
- generation of a filled/void volumetric report

#### Outputs from `Code_2.py`

Typical outputs include:

- combined heterogeneous ballast domain in HDF5 format
- 2D projection image of the assembled volume
- volumetric report with filled and void percentages
