# Voxelized_Railway_Ballast_Generation

This repository contains the Python implementation used to generate and assemble heterogeneous voxelized railway ballast models for ground-penetrating radar (GPR) simulations.

The repository provides a reproducible workflow for constructing realistic ballast geometries and preparing them for numerical modeling.

## Associated publication

This repository accompanies the manuscript:

## Repository structure

### Code
Python scripts used for generating and assembling voxelized ballast models.

- `Code/Code_1.py` — generation of voxelized ballast sub-blocks  
- `Code/Code_2.py` — assembly of the full heterogeneous ballast domain  

### Input
3D stone geometry files (`.obj`) used to construct ballast particles.

### Output
Example outputs generated by the code, including:

- voxel grids (PKL files)
- 3D voxel visualizations (PNG)
- 2D projection images (PNG)
- HDF5 datasets for simulation input
- summary reports (TXT)

## Description

The workflow consists of two main stages:

### Code_1.py — Generation of voxelized sub-blocks

This script generates **147 independent voxelized ballast sub-blocks** using procedurally placed 3D stone geometries.

Main operations include:

- loading 3D stone geometries from `.obj` files  
- applying stochastic rigid-body rotations  
- systematic placement of stones within a voxel grid  
- overlap-controlled voxel assembly  
- voxelization using Delaunay triangulation  
- generation of per-layer outputs and statistics  

Each generated sub-block has:

- size: **0.1 m × 0.1 m × 0.1 m**  
- voxel resolution: **0.002 m**  

Outputs include:

- voxel grids (`.pkl`)
- HDF5 files (`.h5`)
- 2D projections
- 3D coloured voxel visualizations
- per-layer overlap and fill statistics (`final_report.txt`)

---

### Code_2.py — Assembly of heterogeneous ballast model

This script loads the 147 voxelized sub-blocks and assembles them into a structured:

- **7 × 7 × 3 grid**

to create a full ballast domain of:

- **0.7 m × 0.7 m × 0.3 m**

Main operations include:

- loading HDF5 voxel blocks  
- structured 3D grid assembly  
- boundary correction across internal interfaces  
- generation of 2D projection visualization  
- export of combined HDF5 domain  
- computation of filled/void percentages  

Outputs include:

- combined voxel domain (`.h5`)
- 2D projection image  
- volumetric report (`.txt`)
